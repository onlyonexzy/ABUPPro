## 功能需求

### 1. 登录

#### 1.1 登录入口与首次安装展示
- 需求ID：LG-01
- 需求名称：登录入口与首次安装展示
- 需求描述：首次安装仅展示在线登录与域账号登录入口。
- 业务场景描述：首次安装后进入登录页。
- 前置条件：本地无历史账号缓存
- 触发条件：首次安装进入登录页
- 功能和规则详细说明：
  - 首次安装仅显示【在线登录】与【域账号登录】按钮。
  - 首次安装以本地无历史账号缓存为准。
- 输出描述：登录页显示在线/域账号入口。
- 备注：无
- 假设与待确认：无

#### 1.2 账号记忆与账号列表
- 需求ID：LG-02
- 需求名称：账号记忆与账号列表
- 需求描述：最多存储 5 个账号密码，展示上次登录账号。
- 业务场景描述：用户重复登录与快速输入。
- 前置条件：历史登录记录存在
- 触发条件：进入登录页
- 功能和规则详细说明：
  - 登录页仅显示上次登录账号。
  - 其他账号默认不显示、不下拉。
  - 覆盖策略建议 LRU，最多存储 5 个账号。
- 输出描述：账号记忆与列表展示规则。
- 备注：无
- 假设与待确认：无

#### 1.3 记住密码
- 需求ID：LG-03
- 需求名称：记住密码
- 需求描述：支持记住密码且遵循安全存储。
- 业务场景描述：用户希望减少输入密码操作。
- 前置条件：账号已存在
- 触发条件：勾选“记住密码”
- 功能和规则详细说明：
  - 在线登录校验完全交给云端。
  - 勾选：下次免输密码。
  - 未勾选：不展示密码，但本地仍存储密码与权限。
  - 密码必须安全存储（禁止明文）。
- 输出描述：记住密码行为与安全约束。
- 备注：无
- 假设与待确认：合规要求需确认

#### 1.4 在线账号密码登录校验
- 需求ID：LG-04
- 需求名称：在线账号密码登录校验
- 需求描述：提供本地空值校验与云端校验提示。
- 业务场景描述：在线登录输入账号密码。
- 前置条件：网络在线
- 触发条件：点击在线登录
- 功能和规则详细说明：
  - 未填写：提示“请填写账号”“请填写密码”。
  - 账号不存在：提示“账号不存在”。
  - 账号密码不匹配：提示“账号或密码输入错误”。
  - 账号被禁用：提示“您的账户已被禁用！”。
  - 无工具端权限：提示“无权限登录”。
  - 连续错误 5 次锁定 10 分钟，锁定期不可登录。
  - 在线锁定状态以云端返回为主，本地仅用于提示展示与缓存剩余时间。
  - 除锁定规则外不限制登录次数。
- 输出描述：登录校验提示与锁定反馈。
- 备注：无
- 假设与待确认：无

### 2. 基础功能

#### 2.1 整体样式说明
- 需求ID：BF-01
- 需求名称：整体样式说明
- 需求描述：统一工具端整体样式与布局规范。
- 业务场景描述：工具端主界面展示与交互。
- 前置条件：进入工具主界面
- 触发条件：UI 设计与实现阶段
- 功能和规则详细说明：
  - 功能设计思路：
    - 顶部区域采用左右中三段式布局，保持信息层级清晰：左侧承载品牌与登录信息（长城图标+用户名），用户名右侧承载高频功能入口（如新建/打开/保存/导出），中间居中展示“工程文件名称-工具名称”，右侧遵循 Windows 原生窗口控制规范（最小化/最大化/关闭），确保一致的桌面应用体验。
    - 左侧功能列表采用二级菜单结构，一级为功能模块分组（控制台/统一刷写/系统设置/诊断管理等），二级为具体页面；交互上支持展开/收起与当前项高亮，保证导航效率与可扫读性。
    - 右侧为多窗口工作区，支持多功能页并行展示；窗口可拖拽、缩放、置顶与切换，提升复杂任务的并行效率，避免单页切换带来的上下文丢失。
  - 交互与状态：
    - 顶部：常用功能按钮提供 hover/disabled 状态；工程文件名超长时省略号+悬浮完整展示；窗口控制按钮使用系统级视觉与交互反馈。
    - 顶部：工具名称旁增加连接指示灯（灰色未连接/绿色连接成功/红色连接失败）。
    - 左侧：一级菜单点击展开/收起，箭头动效提示；二级点击后打开对应功能窗口并高亮；支持滚动。
    - 右侧：窗口具有激活边框与层级关系，提供最小化/最大化/关闭控制；空态可展示指引文案。
  - 视觉规范：
    - 顶部高度 48–56px，浅色背景+底部分割线；字体 12–14px，图标 16px。
    - 侧边栏宽度建议 220–260px，一级标题加粗，二级缩进与主色高亮。
    - 多窗口区域使用卡片式容器，圆角 8–12px，标题栏与内容区分明。
- 输出描述：形成统一的布局与视觉规范说明。
- 备注：无
- 假设与待确认：无

##### 2.2.4 在线更新
- 需求ID：BF-05
- 需求名称：在线更新
- 需求描述：进入在线更新流程。
- 业务场景描述：用户手动检查更新。
- 前置条件：登录成功
- 触发条件：点击“在线更新”
- 功能和规则详细说明：
  - 多窗口展示：点击在线更新后，打开更新窗口。
  - 展示内容与结构：
    - 版本信息 Tab
    - 更新内容 Tab
  - 版本信息：
    - 展示当前版本，格式：当前版本：xxxxxxxxxx
    - 最新版本在未检查更新前显示“---”
    - 点击【检查更新】向云端获取比当前版本新的版本
      - 云端返回新版本后，展示发现的新版本（仅下发最新三个版本，旧版本不再下发）
      - 若无更新版本，展示“已是最新版本”
      - 若存在一个最新版本，默认选中，可立即更新并展示更新进度
      - 若存在多个最新版本，支持单选，选中后可立即更新并展示更新进度
      - 更新失败/成功需展示对应图标与进度，失败后允许重新更新
  - 更新内容：
    - 点击“更新内容”Tab，展示【检查更新】后新版本的更新内容；若未点击【检查更新】，默认空态
    - 展示形式：版本号 + 更新内容，支持下拉
      - 下拉展示该版本更新内容，内容由云端下发“.vi”文件决定（类似 XML 格式）
      - 支持展示多个版本的更新内容（解析云端返回更新文件后展示）
    - 更新内容需包含中英文，切换中英文时同步切换
  - 更新完成后需退出软件，返回登录界面
- 输出描述：更新窗口的版本信息与更新内容展示、更新进度反馈与完成退出逻辑。
- 备注：无
- 假设与待确认：更新回滚策略需确认

##### 2.2.5 修改密码
- 需求ID：BF-06
- 需求名称：修改密码
- 需求描述：进入修改密码流程。
- 业务场景描述：用户调整密码。
- 前置条件：登录成功
- 触发条件：点击“修改密码”
- 功能和规则详细说明：
  - 业务要素：
    - 当前密码（密码输入框）
    - 新密码（密码输入框）
    - 确认密码（密码输入框）
    - 【取消】【确定】按钮
  - 校验规则：
    - 密码输入框为特例输入框
    - 输入文本以 * 代替明文（默认提示文本除外）
    - 不允许剪切/复制/粘贴
    - 长度 8-14 个字符
    - 不允许 4 位连续相同字符（如 0000/AAAA）
    - 不允许 4 位连续递增或递减的数字或字母（如 1234/4321/ABCD/DCBA）
    - 必须包含大小写字母、数字、特殊字符中的 3 种组合
    - 不允许空格、中文
    - 与确认密码不一致提示：“两次输入密码不一致”
  - 点击【确定】后通过云端接口修改密码：
    - 云端返回成功：保存新密码；点击【立即重启】或“x”均退出登录并回到登录界面
    - 云端返回失败：不保存新密码，不退出登录，展示失败提示语
  - 点击【取消】或“x”放弃本次重置密码
  - 密码格式提示固定展示在下方
    - 长度为 8-14 个字符
    - 不能包含 4 位连续相同的字符，如 0000 或 AAAA
    - 不能包含 4 位连续递增或递减的数字或字母，如 1234 或 4321 或 ABCD 或 DCBA
    - 包含大小写字母和数字、特殊字符里的 3 种组合
    - 不允许有空格、中文
- 输出描述：修改密码的输入要素、校验规则与提交结果处理。
- 备注：无
- 假设与待确认：无

#### 4.1 GBF转化
- 需求ID：GDT_GBF_001
- 需求名称：GBF转化
- 需求描述：支持将原始刷写文件或压缩包转化为 GBF 文件，并支持整车打包。
- 业务场景描述：供应商或长城内部人员导入本地刷写包，或从云端同步刷写包后，完成 Header 配置、单个/批量转化与整车打包。
- 前置条件：系统正常使用，且拥有GBF转化权限。
- 触发条件：点击【统一刷写】->【GBF转化】功能菜单。
- 功能和规则详细说明：
  - 展示 GBF 转化主窗体，左侧为文件操作栏与文件列表，右侧为文件结构、解析结果与 GBF Header 字段配置区。
  - 顶部展示【添加文件】【添加文件夹】【从云端同步】【整车打包】【转化】按钮，按钮启用状态随选中文件、文件类型和配置状态联动。
  - 左侧列表底部展示总体转化进度文字，并支持打开“总体进度”视图。
  - 原型图中需覆盖空态、已导入、待配置、配置完成、转化中、转化成功、转化失败等状态。
  - GBF 文件结构遵循《GBF 定义 V1.0.6》：`.gbf` 文件由 Header、可选 OTAHeader 和 DATA 三部分组成；其中 Header 为 ASCII 编码 JSON 文本，OTAHeader 和 DATA 为二进制内容；Header 以 `}` 结束，结束后直接进入二进制数据，不使用额外分隔符。
  - 34 刷写文件的 DATA 部分按 Block 组织；38 刷写文件的 DATA 部分按 File 组织；右侧解析区需根据 flashType 展示对应结构。
  - 右侧配置区或配置弹框中的字段内容需基于 GBF Header 定义分组展示，至少包含以下分组：
    - 基础信息：`gbfVersion`、`gdtVersion`、`busType`、`flashType`、`supplierCode`。
    - ECU 通讯参数：`ecuEthComParam` 或 `ecuCanComParams`，两者互斥。
    - ECU 版本信息：`ecuVersion.hwVersion`、`ecuVersion.swVersion`、`ecuVersion.baselineVersion`。
    - 安全算法：`sas.saType`、`sas.saLvl`、`sas.saMask`，支持多组。
    - 扩展结构：`otaHeader`、`dataContent`、`flashFile38s`、`flashData34.blocks`、`ecuSwSignature`、`hutSshfsInfo`。
  - 字段展示需区分三类状态：用户可编辑字段、工具自动计算只读字段、预留字段。预留字段默认折叠或隐藏，仅在需要时展开。
  - `gbfVersion` 和 `gdtVersion` 由工具自动填写；版本号格式统一为 `Vxx.xx.xx`。
  - `busType` 为必填单选项，支持 CAN、CANFD、Ethernet 等枚举；`flashType` 为必填单选项，且需与 `busType` 联动：Ethernet 仅展示 Type I/Type II，CAN/CANFD 仅展示 CAN 非压缩/压缩类型。
  - `supplierCode` 为必填字段，需校验 ASCII 格式，长度不超过 15 个字符。
  - 当 `busType=Ethernet` 时，展示并要求填写 `ecuEthComParam`；当 `busType=CAN/CANFD` 时，展示并要求填写 `ecuCanComParams`；未命中的通讯参数分组不展示。
  - 以太网通讯参数需展示 `ecuName`、`logicAddress`、`funId`、`ipAddress`；其中 `funId` 默认 `0xE400`，逻辑地址与功能地址需校验 HEX 格式并在生成文件时自动补齐到 4 字节。
  - CAN/CANFD 通讯参数需展示 `ecuName`、`requestId`、`responseId`、`logicAddressForXota`、`funId`；其中 `funId` 默认 `0x7DF`，并支持多组 ECU 地址配置。
  - ECU 版本信息中的 `hwVersion`、`swVersion`、`baselineVersion` 均为必填；其中 `hwVersion`、`swVersion` 需校验 ASCII 格式。
  - 安全算法区需支持多组 SA 配置；`saMask` 长度需与 `saType` 联动，4 字节算法使用 4 字节掩码，16 字节算法使用 16 字节掩码。
  - `otaHeader` 区域用于上传 OTA Bin 文件；当 `flashType=2`（38 服务刷写）时该区域不展示；其他刷写类型下可展示为可选上传区。
  - `otaHeader.gbfOtaOffset`、`otaHeader.length`、`otaHeader.checkSum` 以及 `dataContent` 中的地址、长度、校验和由工具自动计算，不允许手工编辑，可作为只读信息展示或默认隐藏。
  - 38 服务结构区需展示 `installAddress`、`fileName`、`dataType`、`fileSizeUnzip`、`fileSizeZip`、`fileHash`；其中仅安装路径允许手工编辑，其余由工具自动提取或计算。
  - 34 服务结构区需展示 `swType`、`dataType` 和 `blocks` 列表；`blocks` 至少展示 `blockDataType`、`fileIndex`、`gbfBlockOffset`、`startAddress`、`length`、`checkSum`。
  - `blocks.startAddress` 与 `length` 在 Hex/S19/FP 场景下优先自动解析，在 BIN/OTA BIN 场景下支持手工补录；所有 HEX 类型字段页面允许简写显示，生成 GBF 时统一补齐字节位数并自动补 `0x` 前缀。
- 输出描述：提供 GBF 转化主入口、文件操作区、配置区和进度反馈区。
- 备注：支持的原始文件格式以压缩包内内容为准，包含 S19、Hex、BIN、OTA BIN、FP 等类型。
- 假设与待确认：字段最终命名以 GBF 定义正式版为准，预留字段是否默认展示、BIN/OTA BIN 地址补录方式待最终确认。

- 需求名称：转化包状态

- 业务场景描述：用户导入刷写包后，需要快速判断当前文件是否已完成配置、是否可以发起转化、当前是否正在执行、以及最终是否成功或失败。

  - 配置未完成：展示红色点，表示当前文件仍存在必填项未完成、校验未通过，或虽已通过校验但尚未勾选【完成配置】。

  - 转化失败：展示红色 `×`，表示转化执行失败，失败原因需在详情区和系统消息中同步展示，并允许用户修正后重新发起转化。

- 仅“配置完成”状态的 `.zip` 文件可点击【转化】；`.gbf` 文件不参与再次转化，仅支持查看、校验和整车打包。

    - 每个文件的输出目录与原文件目录保持一致。
    - 若某文件在 Header 校验、地址补齐、CRC32 计算、Hash 计算或 34/38 结构组装阶段失败，应标记该文件
    失败并输出具体错误信息到系统消息。
    -转化完成的数据包，需要在列表中显示。

##### 4.2.1 GBF数据导入解析
- 需求ID：GDT_ECUFLASH_002
- 需求名称：刷写配置-GBF数据导入解析
- 需求描述：导入GBF后，支持对Header数据进行解析，并自动加载和生成总线、ECU、Header中完整的信息。
- 业务场景描述：用户在执行单件刷写前，需要先导入 GBF、TB2、SA算法等数据；系统解析后生成当前 ECU 的刷写上下文。
- 前置条件：
  - 有统一刷写菜单权限。
  - 成功登录并进入【刷写配置】页面。
- 触发条件：
 - 点击【刷写配置】-【文件导入】
 - 【刷写配置】-【文件夹导入】
 - 云端下载GBF文件
- 功能和规则详细说明：
  - 仅支持.gbf和.zip文件的导入；仅支持.gbf文件的解析。
  - 导入可以导入.zip文件，自动解压后解析gbf文件，仅支持一层zip的解压；若zip中还有其他格式文件，均无需导入和解析。
  - 数据解析成功后，需自动回填以下内容：
    - 总线信息：总线名称、busType对应的总线类型，flashType对应的刷写类型；（具体内容见总线配置）
    - ECU信息根据不同的总线类型，对应不同的ECU信息；（具体内容见ECU配置）
      - 以太网ECU：供应商、ECU名称、逻辑地址、功能地址、IP地址
      - CANECU：供应商、ECU名称、请求地址、响应地址、功能地址、XOTA逻辑地址
  - 导入时，若导入同一个ECU（供应商编码），需要新建总线，将ECU放入新的新的总线下，也就是同一个总线下不能有相同的ECU（供应商编码）；不同的gbf存在相同总线时，分布在相同总线下；默认新建总线：CAN1，CANFD1，ETH1；
  - 解析失败时，需明确提示失败原因，且不得覆盖用户已确认保存的旧配置。
  - 解析成功后，将GBF的中内容填充到对应的ECU和总线中。
- 输出描述：提供GBF的数据导入、结构解析和配置自动回填能力。
- 备注：所有信息均来自GBF，一些GBF中不存在的信息，需要固定填充默认值。
- 假设与待确认：不同供应商包格式差异导致的解析容错策略待确认？--相同ECU不同供应商，按照同一个ECU处理；

##### 4.2.2 总线配置
- 需求ID：GDT_ECUFLASH_003
- 需求名称：添加和编辑总线内容
- 需求描述：用户可以手动添加不同类型的总线，以及编辑总线的信息；
- 业务场景描述：用户在执行单件刷写前，需要先配置总线或者使用默认的总线信息。
- 前置条件：
  - 有统一刷写菜单权限。
  - 成功登录并进入【刷写配置】页面。
- 触发条件：点击【添加总线】的按钮；点击具体总线，右侧展示总线的数据可以进行编辑。
- 功能和规则详细说明：
  - 总线类型和总线名称均唯一，总线类型总共只有CAN，CANFD，ETH，LIN四种；总线名称，唯一不可重复。
  - 自动添加：单个导入/批量导入时，从gbf文件中读取ECU的总线信息，自动形成总线信息。
    - 未创建任何总线，导入时根据总线类型，自动添加总线；比如导入的总线类型为CANFD，则自动添加CANFD1；例：导入4个文件，文件1总线类型CAN，文件2总线类型CANFD，文件3总线类型ETH，文件4总线类型CANFD，则自动创建CAN1，CANFD1，ETH1的总线，且CAN1下存在一个ECU，CANFD1下存在两个ECU，ETH1下存在一个ECU；
    - 已存在一个或多个总线，根据总线类型，自动自动加入到已存在的总线中。
  - 手动添加：点击【添加总线】按钮，悬浮框显示添加的总线的类型，选择4个类型中的一个，点击后可直接添加总线；总线名称为类型后面默认加1,2,3的序号作为唯一的名称。
  - 总线编辑：点击具体的总线，右侧展示总线的信息；
   -修改和保存按钮；点击可以进行编辑，点击保存，保存数据。
   - CAN总线信息
     - 可编辑信息：总线名称；标准波特率，可下拉选择100kbps，125kbps，250kpbs，500kpbs，1Mkpbs，默认为500kbps；
     - 不可编辑：总线类型，采样点，Tq，时间量，预定标器，位定时段1，位定时段2，同步跳转宽度；
   - CANFD总线信息
     - 可编辑信息：总线名称；标准波特率，可下拉选择100kbps，125kbps，250kpbs，500kpbs，1Mkpbs，默认为500kbps；数据波特率，可下拉选择2Mkpbs，4Mkpbs，6Mkpbs，默认为2Mbps；
     - 不可编辑：总线类型，采样点，Tq，时间量，预定标器，位定时段1，位定时段2，同步跳转宽度；
   - ETH总线信息
     - 可编辑信息：总线名称；波特率，可下拉选择100Mkpbs，1000Mkpbs，默认为100Mbps；连接类型：下拉直连 或 OBD转以太网，默认OBD；
     - 不可编辑：总线类型
   - LIN总线信息
     - 可编辑信息：总线名称；波特率，可下拉选择2.4kpbs，9.6kpbs，19.2kpbs，默认为19.2kbps；
     - 不可编辑：总线类型     
  - 导入失败时，需要在错误信息中添加错误原因，比如：xxxxx.gbf 文件解析失败；
- 输出描述：自动或手动形成总线，总线信息可以被修改和保存。
- 备注：暂无
- 假设与待确认：ETH和LIN的属性待确认

    - 自选算法允许从本地导入 DLL 文件。

  - 算法类型和掩码来源于 GBF 解析结果。
  - 当选择默认算法时，执行默认算法 DLL 与 GBF 掩码；当选择自选算法时，不再使用 GBF 掩码。

- 假设与待确认：云端默认算法的版本控制与缓存策略待确认。

###### 4.2.3.3 刷写策略

### 6. 系统设置

#### 6.1 统一刷写设置
- 需求ID：GDT_SET_001
- 需求名称：统一刷写设置
- 需求描述：配置统一刷写中的默认上传方式、DID 读取策略、数据更新和文件存储路径。
- 业务场景描述：用户在执行 GBF 转化、单件刷写、整车刷写前，先设置统一刷写的全局默认策略。
- 前置条件：具备系统设置菜单权限。
- 触发条件：点击【设置】->【统一刷写设置】菜单。
- 功能和规则详细说明：
  - 配置项需全局生效，并影响 GBF 转化、单件刷写、整车刷写中的默认行为。
  - 设置页至少包含：日志上传策略、单件刷写前读取 DID 配置、整车刷写读取 DID 配置、统一刷写数据更新、统一刷写文件存储路径。
  - 统一刷写设置修改后应立即作用于后续执行流程，不要求用户重新登录。
- 输出描述：提供统一刷写相关的全局策略配置能力。
- 备注：统一刷写设置与驱动安装属于不同子页，不应混在同一张表单中。
- 假设与待确认：设置项的本地持久化形式和权限边界待确认。

##### 5.1.1 日志上传策略
- 需求ID：GDT_SET_001_01
- 需求名称：统一刷写设置-日志上传策略
- 需求描述：配置单件刷写和整车刷写中的日志上传方式与刷写包上传策略。
- 业务场景描述：用户希望统一控制日志上传时机和是否附带刷写包。
- 前置条件：进入【统一刷写设置】页面。
- 触发条件：打开设置页或修改上传配置。
- 功能和规则详细说明：
  - 单件刷写：
    - 日志上传方式支持【手动上传日志】/【自动上传日志】。
    - 刷写包上传策略支持【不上传刷写包】/【上传刷写包】。
  - 整车刷写：
    - 日志上传方式支持【手动上传日志】/【自动上传日志】。
    - 刷写包上传策略支持【不上传刷写包】/【上传刷写包】。
  - 默认值均为【手动上传日志】+【不上传刷写包】。
  - 当单件刷写配置为自动上传时，执行页不再展示手动上传按钮。
- 输出描述：形成统一刷写中的默认上传策略。
- 备注：整车刷写当前执行页仍以手动上传为主，但设置页需保留策略位。
- 假设与待确认：自动上传失败后的重试策略待确认。

##### 5.1.2 单件刷写前读取DID配置
- 需求ID：GDT_SET_001_02
- 需求名称：统一刷写设置-单件刷写前读取DID配置
- 需求描述：配置单件刷写前默认读取的 DID。
- 业务场景描述：单件刷写前，系统需要根据默认 DID 读取 ECU 当前版本和序列号。
- 前置条件：进入【统一刷写设置】页面。
- 触发条件：点击新增、删除或修改 DID 配置。
- 功能和规则详细说明：
  - 默认内置以下 DID：
    - 应用软件版本：`22F189`，固定解析。
    - 标定软件版本：`22F1C0`，固定解析。
    - 底层软件版本：`22F1C1`，固定解析。
    - ECU 序列号：`22F1xx`，固定解析。
  - 支持新增 DID 和删除 DID。
  - 点击【添加】后，弹框输入 DID，仅支持 4 位十六进制校验，输入错误时给出提示。
  - 添加成功后，列表增加一条读取项，前缀自动补为 `22 + DID`。
  - 勾选后，点击【删除】进行移除；删除前弹出二次确认：“确认需要移除这些数据？”。
  - 自行新增的 DID 无需固定解析，直接将 `62` 之后的报文作为数据上报。
- 输出描述：形成单件刷写前版本读取与序列号读取的 DID 配置表。
- 备注：默认 DID 应和单件刷写中的版本读取逻辑强绑定。
- 假设与待确认：`22F1xx` 的最终展开值和序列号解析规则待确认。

##### 5.1.3 整车刷写读取DID配置
- 需求ID：GDT_SET_001_03
- 需求名称：统一刷写设置-整车刷写读取DID配置
- 需求描述：配置整车刷写前默认读取的逻辑地址与 DID。
- 业务场景描述：整车刷写前，需要通过网关或指定逻辑地址读取 VIN 等整车级信息。
- 前置条件：进入【统一刷写设置】页面。
- 触发条件：点击新增、删除或修改整车 DID 配置。
- 功能和规则详细说明：
  - 默认可预置读取网关 VIN 的一条记录，具体值待确认。
  - 支持新增逻辑地址和 DID：
    - 逻辑地址与 DID 均必须为 4 位十六进制。
    - 输入错误时需要即时提示。
  - 支持勾选后删除，删除前弹出二次确认：“确认需要移除这些数据？”。
- 输出描述：形成整车刷写前的逻辑地址 + DID 读取配置表。
- 备注：读取结果应服务于整车刷写中的【获取ECU版本】。
- 假设与待确认：VIN 默认读取对象和 DID 待最终确认。

##### 5.1.4 统一刷写数据更新
- 需求ID：GDT_SET_001_04
- 需求名称：统一刷写设置-数据更新
- 需求描述：检查并更新统一刷写所依赖的数据版本。
- 业务场景描述：用户希望确认本地统一刷写数据是否为最新，并可一键更新。
- 前置条件：进入【统一刷写设置】页面且具备联网条件。
- 触发条件：点击【检查数据】或【立即更新】。
- 功能和规则详细说明：
  - 默认展示当前数据版本；登录时也需要校验此版本。
  - 点击【检查数据】后，通过接口获取云端最新数据版本并展示。
  - 当云端版本与当前版本相同时，【立即更新】不可点击。
  - 当云端版本与当前版本不同时，允许点击【立即更新】拉取最新版本，并删除当前数据包。
- 输出描述：提供统一刷写数据的版本检查与更新能力。
- 备注：版本更新不应阻塞设置页的其它编辑项。
- 假设与待确认：数据更新失败回滚策略待确认。

##### 5.1.5 统一刷写文件存储路径
- 需求ID：GDT_SET_001_05

- 需求描述：配置统一刷写下载文件、日志文件等的默认存储目录。
- 业务场景描述：用户希望统一控制云端下载包、日志和导出文件的本地落盘路径。
- 前置条件：进入【统一刷写设置】页面。
- 触发条件：点击路径选择按钮或保存路径修改。
- 功能和规则详细说明：
  - 支持选择文件夹，配置默认下载路径和日志文件存储地址。
  - 路径配置完成后，应同步影响：
    - GBF 转化中的云端下载默认路径。
    - 单件刷写的日志导出/上传暂存路径。
    - 整车刷写的日志导出/上传暂存路径。
  - 路径无效或不可写时，需要给出即时错误提示。
- 输出描述：形成统一刷写相关文件的默认存储路径配置。
- 备注：路径修改应立即生效，无需重新登录。
- 假设与待确认：下载目录与日志目录是否拆分成两个独立路径待确认。
